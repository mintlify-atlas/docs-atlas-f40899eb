## Get Started in 5 Minutes

This guide will help you clone the repository, start the application, and see your first property management dashboard.

<Steps>
  <Step title="Clone the Repository">
    Clone the Inmobiliaria Web project to your local machine:

    ```bash
    git clone <url-del-repositorio>
    cd inmobiliaria_web
    ```

    The project structure includes:
    - `index.html` - Main application entry point
    - `pages/` - Individual page components (home, productos, categorías)
    - `JS/` - JavaScript modules for API integration and UI logic
  </Step>

  <Step title="Start the Development Server">
    Use Live Server in VS Code to serve the application:

    <Note>
      **VS Code Live Server Extension Required**
      
      Install the Live Server extension from the VS Code marketplace if you haven't already.
    </Note>

    1. Open the project folder in VS Code
    2. Right-click on `index.html`
    3. Select **"Open with Live Server"**

    The application will open automatically at `http://127.0.0.1:5500` (or similar port).
  </Step>

  <Step title="Configure the Backend">
    <Warning>
      The application requires a backend API running at `http://localhost:8080`
    </Warning>

    Ensure your backend server is running and accessible. The frontend expects these endpoints:

    ```javascript
    // Product endpoints
    GET  http://localhost:8080/api/producto/listar?page=0&size=20
    POST http://localhost:8080/api/producto/registrar
    PUT  http://localhost:8080/api/producto/actualizar/{id}
    DELETE http://localhost:8080/api/producto/{id}

    // Category endpoints  
    GET  http://localhost:8080/api/categoria/listar?page=0&size=20
    GET  http://localhost:8080/api/categoria/listar-nombre
    POST http://localhost:8080/api/categoria/registrar
    PUT  http://localhost:8080/api/categoria/actualizar/{id}
    DELETE http://localhost:8080/api/categoria/{id}
    ```

    If your backend is running on a different port, see [Installation](/installation#backend-configuration) for configuration options.
  </Step>

  <Step title="Verify the Application">
    Once the application loads, you should see:

    - **Navigation bar** with Home, Productos, and Categorías sections
    - **Dashboard** displaying property statistics:
      - Total products count
      - Total categories count
      - Inventory value
      - Average price
      - Most expensive/cheapest products
      - Distribution by category

    <Note>
      If you see empty data or errors, verify that your backend is running and returning data correctly.
    </Note>
  </Step>
</Steps>

## Quick Example: Fetching Products

Here's how the application fetches products from the backend:

```javascript
function obtenerProductos(pagina = 0) {
  let url = `http://localhost:8080/api/producto/listar?page=${pagina}&size=20`;

  // Add filters to URL if they have values
  const nombre = document.getElementById("filtroNombre")?.value.trim();
  if (nombre) {
    url += `&nombre=${encodeURIComponent(nombre)}`;
  }

  fetch(url)
    .then((response) => response.json())
    .then((data) => {
      const lista = Array.isArray(data) ? data : data.content || [];
      productosGlobales = lista;
      paginaActual = data.number || 0;
      totalPaginas = data.totalPages || 1;
      
      renderizarProductos(productosGlobales);
      actualizarControlesPaginacion();
    })
    .catch((error) => console.error("Error:", error));
}
```

This function demonstrates:
- Pagination with 20 items per page
- Dynamic URL parameter building
- Handling both array and paginated responses
- Error handling with catch block

## Troubleshooting

<AccordionGroup>
  <Accordion title="Application shows 'Failed to fetch' errors">
    This usually means the backend is not running or not accessible at `http://localhost:8080`.

    **Solutions:**
    1. Verify your backend server is running
    2. Check the backend port matches (default: 8080)
    3. Look for CORS errors in browser console
    4. Ensure backend allows requests from your Live Server origin
  </Accordion>

  <Accordion title="Dashboard shows all zeros">
    The dashboard displays data from the backend. Empty values mean:
    - Backend has no data in the database
    - Backend API is not returning data correctly

    **Solutions:**
    1. Add test data to your backend database
    2. Check backend logs for errors
    3. Verify API responses in browser Network tab
  </Accordion>

  <Accordion title="Live Server not working">
    **Solutions:**
    1. Install Live Server extension from VS Code marketplace
    2. Try a different port if 5500 is in use
    3. Use an alternative like `python -m http.server` or `npx serve`
  </Accordion>
</AccordionGroup>
