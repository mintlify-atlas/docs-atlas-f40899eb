## Overview

The Inmobiliaria Web application follows a modular structure with clear separation between HTML pages, JavaScript modules, and CSS stylesheets. The project uses a simple iframe-based navigation system with Bootstrap 5 for styling.

<Accordion title="JS/loadPage.js - Navigation Handler" icon="arrow-right-arrow-left">

Simple page loading mechanism using iframe:

```javascript JS/loadPage.js
const contentFrame = document.getElementById("contentFrame");

const loadPage = (url) => {
  contentFrame.src = url;
};
```

This allows navigation without full page reloads, maintaining the header and application state.

<Accordion title="JS/sortHelper.js - Sorting Utility" icon="arrow-up-arrow-down">

Reusable sorting helper class for table columns:

```javascript JS/sortHelper.js
class SortHelper {
  constructor(campoInicial = "codigo") {
    this.ordenActual = campoInicial;
    this.direccionActual = "asc";
  }

  ordenarPor(campo, callback) {
    if (this.ordenActual === campo) {
      this.direccionActual = this.direccionActual === "asc" ? "desc" : "asc";
    } else {
      this.ordenActual = campo;
      this.direccionActual = "asc";
    }
    if (callback) callback();
  }

  getParams() {
    return `sort=${this.ordenActual},${this.direccionActual}`;
  }

  actualizarIconos(campos) {
    campos.forEach((campo) => {
      const iconElement = document.getElementById(`icon-${campo}`);
      if (!iconElement) return;

      if (campo === this.ordenActual) {
        if (this.direccionActual === "asc") {
          iconElement.innerHTML = '<i class="bi bi-arrow-up"></i>';
        } else {
          iconElement.innerHTML = '<i class="bi bi-arrow-down"></i>';
        }
      } else {
        iconElement.innerHTML = '<i class="bi bi-arrow-down-up"></i>';
      }
    });
  }
}
```

**Features:**
- Toggle sort direction on same field
- Generate URL parameters for backend sorting
- Update UI icons to indicate sort state

## Design Patterns

### Module Pattern

Each JavaScript file follows a module pattern with:
1. Global state variables at the top
2. API fetch functions
3. Rendering functions
4. Event handlers
5. Initialization code at the bottom

### State Management

<Steps>
  <Step title="Global Arrays">
    Products and categories are stored in global arrays (`productosGlobales`, `categoriasGlobales`)
  </Step>
  
  <Step title="SessionStorage">
    Edit forms use `sessionStorage` to pass data between pages without URL parameters
  </Step>
  
  <Step title="Pagination State">
    Current page and total pages tracked in module-level variables
  </Step>
</Steps>

### Error Handling

All API calls include comprehensive error handling:
- Try-catch blocks for network errors
- Response validation
- User-friendly error messages
- Field-level validation error display

<Note>
  **Best Practice:** All fetch calls check `response.ok` and parse error JSON to display specific validation messages to users.
</Note>
